# 🐍 AssetFlow Backend & Systems Infrastructure Roadmap (Developer A)

**Core Stack:** Python 3.14.x, FastAPI 0.139.x, SQLAlchemy 2.0.51, Alembic 1.14.x, Pydantic 2.10.x, Celery 5.4.x, Redis 5.2.x, Psycopg3 3.2.x

---

## 🗓️ Week 1: Foundational Architecture & Core REST Engine

### Days 1 – 2: Environment Setup & Relational Modeling
*   [ ] Initialize the project monorepo and construct a containerized environment via `docker-compose` spinning up local instances of **PostgreSQL** and **Redis**.
*   [ ] Configure a baseline FastAPI application instance incorporating global CORS middleware profiles.
*   [ ] Design the relational core database structures inside SQLAlchemy 2.0 utilizing an explicit Asynchronous Engine context (`AsyncSession`):
    *   `Users` (Credentials, Role enumerations)
    *   `Assets` (Serial numbers, state bounds, soft-delete visibility parameters)
    *   `Locations` (Adjacency-list based structural node trees)
    *   `AuditLogs` (Immutable tracking structures)
*   [ ] Initialize programmatic schema migrations via Alembic using `alembic revision --autogenerate`.

### Days 3 – 4: Auth Infrastructure & Enterprise CRUD
*   [ ] Implement **JWT Token Authentication** routing. Extract and distribute authentication assertions via secure cookies.
*   [ ] Formulate reusable Dependency Injection checks (`Depends`) to protect routes based on **Role-Based Access Control (RBAC)** limits (`Admin` vs. `Employee`).
*   [ ] Build foundational resource endpoints:
    *   `POST /api/v1/assets` (Instantiate asset records)
    *   `GET /api/v1/assets/{id}` (Retrieve deep resource details)
    *   `PATCH /api/v1/assets/{id}` (Execute safe soft-delete procedures by overriding `is_deleted=True`)

### Days 5 – 7: Data Validation & Query Optimization
*   [ ] Build robust incoming and outgoing payload mapping checks using **Pydantic v2** validation models.
*   [ ] Optimize relational loading routines inside the database service tier—leveraging `joinedload` and `selectinload` methods to entirely avoid N+1 query overhead.
*   [ ] Expose and verify the autogenerated interactive OpenAPI specifications via the local `/docs` path.

## 🗓️ Week 2: Advanced Data Logic, Async Systems & Deployment

### Days 8 – 9: Spatial Hierarchies & Data Virtualization
*   [ ] Write a recursive Common Table Expression (CTE) engine or optimized database query wrapper to handle searching inside multi-tiered location parent/child nodes.
*   [ ] Formulate a cursor-based **server-side pagination utility** supporting operational query parameters (`limit`, `offset`, `search`, `sort_by`).

### Days 10 – 11: Async Workers & Automated Event Auditing
*   [ ] Connect the **Celery task runner** with the localized Redis container instance.
*   [ ] Build asynchronous background worker tasks to compute weekly asset financial depreciation curves and send upcoming warranty expiration notifications.
*   [ ] Integrate database-level interception lifecycle listeners (`SQLAlchemy event hooks`) to seamlessly generate unalterable `AuditLogs` entry records whenever any asset modification transaction commits.

### Days 12 – 14: Quality Assurance & Cloud Release
*   [ ] Write a robust integration and API test layout using `pytest` and `pytest-asyncio` targeting critical auth and mutation routes.
*   [ ] Set up a GitHub Actions workflow pipeline to run validation code linting and automated tests on every branch push.
*   [ ] Deploy the live PostgreSQL instance and FastAPI microservice tier onto **Render** or a cloud infrastructure engine.
